# 更新文档任务

你正在代码变更后更新文档。

## 1. 识别变更
- 检查 git diff 或最近的提交以查找修改的文件
- 识别哪些功能/模块被更改
- 注意任何新文件、删除的文件或重命名的文件

## 2. 验证当前实现
**关键**：不要相信现有文档。阅读实际代码。

对于每个更改的文件：
- 阅读当前实现
- 理解实际行为（而非文档中的行为）
- 注意与现有文档的任何差异

## 3. 更新相关文档

- **CHANGELOG.md**：在"Unreleased"部分添加条目
  - 使用分类：Added（新增）、Changed（变更）、Fixed（修复）、Security（安全）、Removed（移除）
  - 使用简洁、面向用户的语言

## 4. 文档风格规则

✅ **简洁** - 为了简洁可以牺牲语法
✅ **实用** - 示例优于理论
✅ **准确** - 代码已验证，而非假设
✅ **最新** - 匹配实际实现

❌ 不要企业级废话
❌ 不要过时信息
❌ 不要未经验证的假设

## 5. 不确定时询问

如果你不确定变更背后的意图或面向用户的影响，**询问用户** - 不要猜测。
